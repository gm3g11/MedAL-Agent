## P0 — Random  _(random)_

**Paper.** Standard active-learning baseline (no single originating paper). Canonically described as the passive/random-sampling control in: Burr Settles, "Active Learning Literature Survey," Computer Sciences Technical Report 1648, University of Wisconsin–Madison, 2009. In the deep-AL / medical-imaging literature it is the universal lower-bound reference against which every acquisition function is compared.

**Algorithm.** Uniform random sampling from the unlabeled pool: at each AL round it draws k pool indices uniformly at random without replacement, ignoring the model, predictions, and features entirely. It is the passive-learning control that an informative acquisition function must beat. There is no scoring step — the policy does not override score() (inherits the base None) and sets needs_pred_cache = False, so no prediction cache or feature matrix is computed for it.

**Acquisition signal.** No acquisition score. Each of the N pool units has equal selection probability k/N; the chosen set is a uniform random k-subset of {0,...,N-1} (sampling without replacement). Effectively score_i = i.i.d. Uniform(0,1) with top-k, i.e. no information from model/probs/features is used.

**Implementation.** File: medal_bench/policies/p0_random.py. RandomPolicy is registered as "P0" via @register("P0"), name="Random", needs_pred_cache=False. It does not implement score(); only select(ctx, scores, k) runs. Steps: (1) n_pool = len(ctx.pool); (2) seed the RNG deterministically as rng = np.random.RandomState(ctx.seed + ctx.round_idx) — note it uses ctx.seed + ctx.round_idx directly, NOT ctx.query_seed, so the stream advances per round; (3) return rng.choice(n_pool, size=k, replace=False).tolist(). The `scores` argument (always None here, since base.score returns None) is ignored. It uses NONE of the _helpers.py functions and reads none of the PolicyContext model/pred_cache/features/valid_bboxes fields — only ctx.pool, ctx.seed, ctx.round_idx. Policy interface is in medal_bench/policies/base.py (PolicyContext dataclass + abstract Policy.select); registry build via medal_bench.policies.build("P0").

**Caveats.** Per-round determinism depends on ctx.round_idx being passed correctly: the seed is ctx.seed + ctx.round_idx, so two rounds with the same (seed, round_idx) yield identical draws and reusing the same round_idx across different intended rounds would repeat selections (acceptable since already-labeled units are removed from the pool between rounds). It deliberately ignores ctx.query_seed (the SeedSequence stream other policies prefer), using the simpler additive seed instead. If k > n_pool, rng.choice(..., replace=False) raises (no internal clamping) — relies on the runner to cap k to the remaining pool. No model/feature/GPU dependencies, so none of the heavier caveats (P2/P9 RAM, P7/P8 SAM-H features, P6 late-round collapse, P9 low-budget predictor failure) apply.

**File.** `medal_bench/policies/p0_random.py`

## P1 — Normalized Entropy (uncertainty sampling)  _(uncertainty)_

**Paper.** Entropy-based uncertainty sampling is a standard active-learning baseline with no single originating paper. The acquisition criterion is Shannon-entropy uncertainty, surveyed in: Burr Settles, "Active Learning Literature Survey," Computer Sciences Technical Report 1648, University of Wisconsin-Madison, 2009 (Section 3.1, "Uncertainty Sampling," entropy variant). The entropy measure itself is from C. E. Shannon, "A Mathematical Theory of Communication," Bell System Technical Journal, 27(3):379-423, 1948. For dense/segmentation AL, per-pixel entropy aggregated over the image is the conventional formulation used across the deep-AL segmentation literature.

**Algorithm.** Uncertainty sampling that ranks unlabeled pool images by predictive entropy of the segmentation model's per-pixel softmax. For each pixel it computes the Shannon entropy of the class distribution, normalizes it to [0,1] by dividing by log C (C = number of classes), then averages the per-pixel normalized entropy over the image's valid (un-padded) region to get one scalar per image. The k images with the highest mean normalized entropy are selected for labeling. Higher score == more uncertain.

**Acquisition signal.** Per-image score = mean over valid-region pixels of H_norm(pixel), where H_norm = (-sum_c p_c * log2 p_c) / log2 C, with p_c the softmax probability of class c at that pixel (C = num_classes). H_norm in [0,1]; for the degenerate C=1 case it falls back to raw H (==0). Selection = top-k images by this mean (descending). The log base cancels in the C>1 ratio; an eps=1e-12 guards log(0).

**Implementation.** Registered as id "P1" -> class EntropyFull (name "Normalized Entropy") in policies/p1_entropy_full.py. It sets needs_pred_cache=True and needs_pred_cache_probs=False, so the runner streams per-batch reductions and never materializes the full (N,C,H,W) probs. Per-pixel score: _helpers.score_per_pixel(probs_b, "normalized_entropy") computes H = sum_c(-p*log2(p+1e-12)) over the class dim and returns H/log2(C) (or raw H if C==1), giving (B,H,W) in [0,1]. Aggregation: _helpers.aggregate(score_map, argmax_b, "valid", valid_bboxes=valid_b) -> _apply_valid slices each sample's (y0,x0,h,w) un-padded rectangle from ctx.valid_bboxes and takes the per-sample mean (full-canvas mean when bbox is None or covers the whole image), avoiding letterbox-pad pixels inflating the score; result is (B,). The runner accumulates these into ctx.streamed_reduce["score"]; score() reads ctx.streamed_reduce (or, in unit tests, falls back to reducing ctx.pred_cache.probs directly via per_batch_reduce on a single whole-pool batch — identical math). finalize_score returns the (N,) score vector. select() = _helpers.topk_indices(scores, k): stable descending argsort, take top-k. Helper files: policies/_helpers.py (score_per_pixel, aggregate/_apply_valid, topk_indices); policies/base.py (Policy interface, PolicyContext with valid_bboxes / streamed_reduce / pred_cache).

**Caveats.** P1 is "Normalized Entropy" (H/log C in [0,1]), not plain mean-over-classes entropy; the legacy "entropy" sub-mode (=H/C) still exists in score_per_pixel for back-compat/ablations and is monotonic with H so it would yield identical rankings, but P1 uses normalized_entropy. Aggregation is over the VALID (un-padded) region via valid_bboxes; if valid_bboxes is None it silently falls back to the full canvas, so letterbox padding can affect scores when bboxes are not supplied. The score is a uniform mean over all pixels (background-dominated), not a foreground/boundary-restricted aggregation. Memory is safe because it uses streaming reduction (needs_pred_cache_probs=False) and never materializes the full (N,C,H,W) probs tensor.

**File.** `medal_bench/policies/p1_entropy_full.py`

## P2 — BALD (MC-dropout)  _(uncertainty)_

**Paper.** Two foundational works combine into this method. The mutual-information acquisition function (BALD = Bayesian Active Learning by Disagreement): Neil Houlsby, Ferenc Huszar, Zoubin Ghahramani, Mate Lengyel, "Bayesian Active Learning for Classification and Preference Learning," arXiv:1112.5745, 2011. The Monte-Carlo-dropout approximation to Bayesian inference used to estimate it: Yarin Gal, Zoubin Ghahramani, "Dropout as a Bayesian Approximation: Representing Model Uncertainty in Deep Learning," ICML 2016, PMLR 48:1050-1059. The combined "Deep Bayesian Active Learning with MC-dropout BALD" recipe: Yarin Gal, Riashat Islam, Zoubin Ghahramani, "Deep Bayesian Active Learning with Image Data," ICML 2017, PMLR 70:1183-1192.

**Algorithm.** BALD selects samples that maximize the mutual information between the model's prediction and its weights, i.e. samples where the individual stochastic models are confident but disagree with each other. Bayesian inference is approximated by Monte-Carlo dropout: T=10 stochastic forward passes are run with dropout layers active at inference, and the per-pixel BALD score is the predictive entropy minus the expected (per-pass) entropy. High score = high epistemic uncertainty (model-disagreement), distinguishing it from plain entropy which also fires on irreducible aleatoric noise.

**Acquisition signal.** Per-pixel BALD = H(E_t[p_t]) - E_t[H(p_t)], where p_t is the softmax prediction on MC pass t (t=1..T), H(p) is the mean-over-classes entropy actually computed in code, H(p) = -mean_c p_c log2 p_c = (1/C) sum_c (-p_c log2 p_c) — a 1/C scaling of standard class-entropy that preserves the per-pixel ranking. Concretely: predictive_entropy = entropy(mean over T passes of probs), mean_per_pass_entropy = mean over T passes of (per-pass entropy); bald_map = predictive_entropy - mean_per_pass_entropy (N,H,W). The per-sample acquisition score is the mean of this map over the valid (un-padded) region. Selection = top-k by descending score.

**Implementation.** In p2_bald.py (class BALD, registered "P2"): (1) seed torch RNG with ctx.seed+ctx.round_idx for deterministic dropout masks; (2) call medal_bench.models.nnunet.enable_mc_dropout(model) which sets only Dropout* modules to train mode while BN/InstanceNorm stay in eval (running stats unperturbed); n_dropouts is logged. (3) Preload all pool images once as (C,H,W) tensors; run T=10 (self.T) MC passes in pass-outer/batch-inner order (mc_batch_size=32) under no_grad. Each batch: p=softmax(model(xb)) (B,C,H,W); e=mean_c(-p*log2 p) per-pass entropy (B,H,W). It accumulates only running_sum_probs (N,C,H,W) and sum_per_pass_entropy (N,H,W) on CPU (no per-pass full copy, ~2x RAM saving) keeping forward/RNG order byte-identical to the naive version. (4) mean_probs=running_sum/T; pred_ent=entropy(mean_probs); mean_pp_ent=sum_per_pass_entropy/T; bald=pred_ent-mean_pp_ent. Both components are written to ctx.diagnostics_out (bald_predictive_entropy, bald_mean_per_pass_entropy, bald_T, bald_n_dropouts_enabled) for the reduces-to-baseline test. (5) model.eval() restores deterministic eval; aggregate(bald, argmax, "valid", valid_bboxes=ctx.valid_bboxes) from _helpers.py reduces (N,H,W)->(N,) via per-sample mean over the un-padded rectangle (falls back to full-canvas mean if bbox is None/full). select() = topk_indices (stable descending sort). Helper files: medal_bench/models/nnunet.py (enable_mc_dropout) and medal_bench/policies/_helpers.py (aggregate / _apply_valid, topk_indices). Note needs_pred_cache=False — BALD runs its own K-pass MC rather than reusing the shared single-pass prediction cache.

**Caveats.** Host-RAM heavy: BALD holds one (N,C,H,W) CPU tensor (running_sum_probs); on high-class datasets (e.g. ext_amos_mri C=16, cholecseg8k C=13) this reaches ~62-84 GB, same regime as P9 — those cells must be routed to high-RAM nodes (H100 ~444 GB). There is currently NO host-RAM preflight (only a VRAM preflight exists), so this routing is manual. Requires the model to have been built with dropout_p>0 (the project nnU-Net default dropout_p=0.1); if dropout is absent, enable_mc_dropout returns 0 toggled modules and BALD degenerates toward plain predictive entropy with disagreement term ~0. The "entropy" used is mean-over-classes scaled (-mean_c p log2 p), which is monotonic with true class-entropy so per-pixel ranking is preserved; it is not the normalized [0,1] entropy used by P1.

**File.** `medal_bench/policies/p2_bald.py`

## P3 — CoreSet (greedy k-center)  _(diversity)_

**Paper.** Ozan Sener and Silvio Savarese. "Active Learning for Convolutional Neural Networks: A Core-Set Approach." International Conference on Learning Representations (ICLR), 2018. (The greedy k-center / farthest-first approximation used here is the paper's robust k-Center / "k-Center-Greedy" algorithm.)

**Algorithm.** A pure representativeness/diversity (coverage) acquisition: it picks the batch of pool points that best "covers" the feature manifold relative to the already-labeled set. It runs greedy k-center (farthest-first traversal): starting from the labeled set as initial centers, it iteratively adds the pool point whose distance to its nearest already-selected/labeled point is maximal, repeating k times. There is no uncertainty/model-confidence term — selection is geometric, in the task U-Net's encoder-bottleneck feature space. The same k-center routine is reused by P5 (entropy→CoreSet hybrid) and by P7 on SAM foundation features.

**Acquisition signal.** argmax_{i in pool} min_{s in labeled∪selected} ||f(x_i) - f(x_s)||_2, greedily for k steps; f = L2-normalized U-Net bottleneck features, Euclidean distance.

**Implementation.** p3_coreset.py CoreSet.select: reads precomputed features ctx.features["task_unet_pool"]/["task_unet_label"] (needs_features=("task_unet",), needs_pred_cache=False — runner precomputes U-Net bottleneck features). L2-normalizes both via local _l2_norm (norm clipped >=1e-12). Concatenates [label_feats; pool_feats], builds full pairwise distance matrix via sklearn pairwise_distances(metric="l2"), then calls kcenter_greedy(dist_mat, init_idx=range(n_label), k=k) in _helpers.py. kcenter_greedy marks labeled rows as centers and for k iters selects argmax over candidates of (min distance to any center), appending each newly chosen global index. select() maps back to pool indices via i - n_label (drops negatives). No RNG -> deterministic.

**Caveats.** Builds a FULL (N_label+N_pool) x (N_label+N_pool) dense pairwise distance matrix in memory — O(M^2) RAM/compute; scales poorly for very large pools (though far cheaper than P2/P9's probs-network RAM). Quality depends entirely on the task U-Net's bottleneck features being meaningful; early rounds with a weak model give weak coverage. kcenter_greedy asserts a non-empty init/labeled set (it cannot cold-start with zero labels — would silently under-select, so it fails loudly). No uncertainty signal at all, so it ignores model confidence by design.

**File.** `medal_bench/policies/p3_coreset.py`

## P4 — BADGE  _(hybrid)_

**Paper.** Jordan T. Ash, Chicheng Zhang, Akshay Krishnamurthy, John Langford, Alekh Agarwal. "Deep Batch Active Learning by Diverse, Uncertain Gradient Lower Bounds" (the method is named BADGE). International Conference on Learning Representations (ICLR), 2020.

**Algorithm.** BADGE selects a batch that is jointly uncertain and diverse by clustering in a "gradient-embedding" space. Each pool image is mapped to the gradient of the cross-entropy loss (w.r.t. the final classification/segmentation head) evaluated at the model's own argmax pseudo-label (no ground truth). The gradient norm encodes uncertainty and the gradient direction encodes example identity; running kmeans++ seeding in this space picks k points that are simultaneously high-magnitude (uncertain) and spread out (diverse).

**Acquisition signal.** Per-image gradient embedding g(x) = concat_c g_c(x), where for head-input features z_{h,w} (D-dim) and softmax p_{c,h,w}: g_c(x) = mean_{h,w} [ (p_{c,h,w} - 1[yhat_{h,w}=c]) * z_{h,w} ], with yhat = argmax_c p (pseudo-label, no GT). Embedding dim = num_classes*D. Acquisition = the k kmeans++ seeds chosen in this C*D-dim space; no scalar per-image score / top-k.

**Implementation.** P4 (`p4_badge.py`, `@register("P4")`, `needs_pred_cache=False`) overrides only `select()` (no `score()`). For each pool index i it calls `canonical_ce_grad_embedding(ctx.model, ctx.pool[i], ctx.num_classes)` in `_badge_grad.py`, which: (1) finds the seg head fresh each call via `_find_seg_head` (last Conv2d with out_channels==num_classes, no id-cache so current-round weights are used); (2) registers a forward_pre_hook to capture head-input features z; (3) runs one `@torch.no_grad()` forward, p=softmax(logits), yhat=argmax(p), onehot=one_hot(yhat); (4) computes the closed-form gradient w=(p-onehot), g=(wf @ zf.t())/HW reshaped to (C*D,) on CPU — NO backprop. Embeddings are stacked to X=(N, C*D); `badge_embedding_dim` is logged; then `kmeanspp_indices(X, k, random_state=ctx.seed+ctx.round_idx)` (`_helpers.py`) calls sklearn `kmeans_plusplus` to return the k seeds (logged as `badge_kmeanspp_seed`). The CE+Dice backprop helper `ce_dice_grad_embedding` (same file) is the ablation P4b, not used by P4. Policy interface in `base.py`.

**Caveats.** Segmentation adaptation of an originally image-classification method: the per-pixel head gradient is mean-pooled over H*W into one (C*D) vector per image (canonical BADGE has no spatial pooling). The gradient uses only the final 1x1 seg-head weights (the head-input z plays the role of penultimate features); if head-input and logit spatial sizes differ, z is nearest-neighbor interpolated to match. Embedding dimension scales with num_classes (C*D), so building X = (N x C*D) and running sklearn kmeans++ has O(N*C*D) memory/compute that grows with high class counts. kmeans++ seeding is stochastic but seeded deterministically by ctx.seed+ctx.round_idx. Per the project STATE, P4 is the canonical CE-only BADGE; the CE+Dice variant is ablation-only id P4b.

**File.** `medal_bench/policies/p4_badge.py`

## P5 — Entropy -> CoreSet  _(hybrid)_

**Paper.** Canonical algorithm (Stage-2 core, "k-center-greedy"): Ozan Sener and Silvio Savarese. "Active Learning for Convolutional Neural Networks: A Core-Set Approach." International Conference on Learning Representations (ICLR), 2018. — The two-stage "uncertainty-filter then diversity-cover" hybrid pattern (filter pool by an informativeness score, then k-center on the survivors) follows Suggestive Annotation: Lin Yang, Yizhe Zhang, Jianxu Chen, Siyuan Zhang, Danny Z. Chen. "Suggestive Annotation: A Deep Active Learning Framework for Biomedical Image Segmentation." MICCAI 2017. The P5 source file carries no explicit citation; it is documented in-code as a "two-stage uncertainty-diversity hybrid" combining P1 (normalized entropy) as the filter with P3 (CoreSet / k-center-greedy) as the selector.

**Algorithm.** A two-stage uncertainty-then-diversity hybrid. Stage 1 (uncertainty filter): score every unlabeled pool image by mean normalized predictive entropy (the same signal as P1) and keep the top filter_ratio*k most-uncertain candidates (clamped to be at least k and at most the pool size). Stage 2 (diversity cover): run k-center-greedy (farthest-first) over the task U-Net feature vectors of just those kept candidates, seeded by the already-labeled set, to pick k images that maximally cover the filtered candidate region. The result is k uncertain-AND-diverse images, avoiding the redundancy of pure top-k entropy.

**Acquisition signal.** Stage 1 score per image = mean over the valid (un-padded) region of per-pixel normalized entropy H_norm = -sum_c p_c log2(p_c) / log2(C) in [0,1] (eps=1e-12 inside the log; C=number of classes). Stage 2 selection criterion = greedy farthest-first: at each of k steps pick the candidate whose minimum L2 distance (metric="l2") to the {labeled set ∪ already-selected} feature points is maximal (max-min coverage). The final acquisition is the set of k images returned by this k-center pass over the entropy-filtered subset; there is no single scalar per image — selection is set-based.

**Implementation.** File: medal_bench/policies/p5_entropy_coreset.py (class EntropyCoreSet, @register("P5")). Declares needs_pred_cache=True, needs_pred_cache_probs=False (streaming reduction so the full (N,C,H,W) probs is never materialized), needs_features=("task_unet",). STAGE 1 / score(): per_batch_reduce delegates to _entropy_valid_batch (imported from p1_entropy_full.py), which calls _helpers.score_per_pixel(probs_b, "normalized_entropy") -> (B,H,W) then _helpers.aggregate(..., mode="valid", valid_bboxes=...) to mean over each sample's un-padded rectangle (letterbox-pad pixels excluded). The runner pre-accumulates these per-batch (N,) reductions in ctx.streamed_reduce; score() reads accum["score"] (or, in unit tests, reduces ctx.pred_cache.probs directly). STAGE 2 / select(): pulls precomputed task U-Net features ctx.features["task_unet_pool"] and ["task_unet_label"] (asserts both present) and calls _helpers.coverage_after_filter_indices(per_sample_score=scores, pool_features, label_features, k, filter_ratio=5.0, metric="l2"). That helper: keep = min(max(ceil(filter_ratio*k), k), n_pool); stable descending argsort of the entropy scores -> keep_idx; concatenates [label_features, pool_features[keep_idx]]; builds a pairwise_distances (sklearn) matrix; runs _helpers.kcenter_greedy(dist_mat, init_idx=range(n_label), k=k) (max-min farthest-first, seeded by the labeled rows, asserts a non-empty init set); then maps the returned filtered-subset indices back to original pool indices. Helper files: medal_bench/policies/_helpers.py (score_per_pixel, aggregate/_apply_valid, coverage_after_filter_indices, kcenter_greedy) and p1_entropy_full.py (_entropy_valid_batch); interface from base.py (PolicyContext, Policy).

**Caveats.** Needs BOTH the prediction cache (for Stage-1 entropy) and precomputed TASK U-Net features for pool+label (Stage-2 k-center); select() asserts task_unet_pool/task_unet_label are present — these are the task-model bottleneck features, NOT SAM-H foundation features (that variant is P7). filter_ratio defaults to 5.0 (the "top-5k entropy" filter); raising it toward the pool size makes P5 collapse toward pure CoreSet (P3), lowering it toward 1 collapses toward pure top-k entropy (P1). k-center-greedy requires a non-empty labeled/init set (kcenter_greedy asserts len(init_idx)>0), so a true cold-start round with no labeled data would fail loudly. The Stage-2 pairwise_distances is O(M^2) over M = n_label + filter_ratio*k filtered candidates, so cost/RAM grow with the labeled set and filter width (much smaller than full-pool CoreSet, but still quadratic in the kept set). Unlike P3, features are NOT L2-normalized before the distance computation here (the metric is raw "l2" on the kept features). Entropy aggregation uses the valid (un-padded) region via valid_bboxes to avoid letterbox-pad pixels inflating scores.

**File.** `medal_bench/policies/p5_entropy_coreset.py`

## P6 — Selective Uncertainty  _(uncertainty)_

**Paper.** Siteng Ma, Haochang Wu, Aonghus Lawlor, Ruihai Dong (University College Dublin), "Breaking the Barrier: Selective Uncertainty-based Active Learning for Medical Image Segmentation," ICASSP 2024 (IEEE International Conference on Acoustics, Speech and Signal Processing), 2024. arXiv:2401.16298. Reference implementation: HelenMa9998/Selective_Uncertainty_AL.

**Algorithm.** Conventional uncertainty AL averages a per-pixel uncertainty over ALL pixels, which in imbalanced segmentation drowns the lesion/organ signal in background. Selective Uncertainty instead FILTERS pixels into two prediction-derived regions — a "target" region (predicted foreground) and a "boundary" region (low-confidence pixels near the decision boundary) — scores each region with its own mean uncertainty, then DIVERSIFIES the query batch by interleaving the two separate rankings. It uses only the per-round prediction cache (no extra forward pass, no MC dropout, no ground-truth masks of unlabeled samples).

**Acquisition signal.** Two independent per-image scores from the softmax probs. Per-pixel uncertainty is normalized predictive entropy H/log2(C) (P1's score_per_pixel, sub="normalized_entropy"). Target region = pixels where fg_prob = 1 - P(class0=background) >= tau_target (default 0.1); boundary region = pixels where margin = top1_prob - top2_prob <= delta_boundary (default 0.2; binary equivalence |p-0.5|<=0.1 <=> margin<=0.2). s_target = mean uncertainty over target pixels; s_boundary = mean uncertainty over boundary pixels (empty mask -> 0, matching the reference's nan_to_num). Selection alternates target-ranking and boundary-ranking (high-uncertainty first), appending unique sample ids until k are chosen. The logged per-image score is 0.5*(s_target + s_boundary), but actual batch selection is the interleave, not top-k.

**Implementation.** In p6_selective_uncertainty.py (class SelectiveUncertaintyAL, registered "P6"). Streaming policy: needs_pred_cache=True, needs_pred_cache_probs=False, so the runner streams per_batch_reduce and never materializes full (N,C,H,W) probs. per_batch_reduce(probs_b, argmax_b, valid_b, offset): (1) unc = score_per_pixel(probs_b, sub="normalized_entropy") from _helpers.py; (2) target_mask = (1 - probs_b[:,0] >= tau_target); (3) margin via torch.topk(probs_b,2,dim=1), boundary_mask = (margin <= delta_boundary); (4) _intersect_valid_ zeroes both masks outside each sample's un-padded rectangle (ctx.valid_bboxes, in place, sliced per-sample) as a hard geometric pad exclusion; (5) _masked_mean gives s_target, s_boundary per sample plus tgt_frac/bnd_frac diagnostics. finalize_score stashes s_target/s_boundary as numpy, emits diagnostics (selu_target_frac, selu_boundary_frac, selu_score_target_mean, selu_score_boundary_mean, selu_valid_intersected), and returns 0.5*(s_target+s_boundary). select(): stable argsort-descending of each component into rank_t and rank_b, then _interleave_rankings alternates a,b,a,b appending unique ids until k. Fallback to combined-score top-k only if score() did not run. Helpers: _helpers.py score_per_pixel; base.py Policy/PolicyContext interface (PolicyContext carries pred_cache, valid_bboxes, streamed_reduce, diagnostics_out).

**Caveats.** P6 shows late-round COLLAPSE in the Stage-2 benchmark: final-round Dice drops sharply on some datasets (care_LA 0.10, hvsmr 0.29) — a real, reproduced finding, not a pipeline bug. Multi-class handling is an undocumented-in-paper adaptation (paper is binary): fg_prob = 1 - P(background), boundary via top1-top2 margin <= 0.2, entropy as the per-pixel metric. Selection is an interleave of two rankings, so the returned per-image score (0.5*(s_target+s_boundary)) is for logging only and does NOT reproduce the batch by top-k. Empty target/boundary masks score 0 (nan->0 per the reference); valid-region intersection is only applied when ctx.valid_bboxes is not None (padded inputs).

**File.** `medal_bench/policies/p6_selective_uncertainty.py`

## P7 — SAM-CoreSet (Foundation-CoreSet)  _(diversity)_

**Paper.** Acquisition algorithm: Ozan Sener and Silvio Savarese, "Active Learning for Convolutional Neural Networks: A Core-Set Approach," International Conference on Learning Representations (ICLR), 2018. (The greedy k-center / farthest-first selection rule.) The feature backbone is SAM: Alexander Kirillov, Eric Mintun, Nikhila Ravi, Hanzi Mao, Chloe Rolland, Laura Gustafson, Tete Xiao, Spencer Whitehead, Alexander C. Berg, Wan-Yen Lo, Piotr Dollar, Ross Girshick, "Segment Anything," ICCV 2023 — used here as a frozen feature extractor (vit_h image encoder), not a published P7-specific method. NOTE: the in-file docstring says "SAM ViT-B"; this is stale. The frozen profile (profiles/frozen.py) and STATE.md pin the canonical extractor to SAM-H / vit_h (sam_vit_h_4b8939.pth); vit_b is permitted only if explicitly configured and produces materially different selections.

**Algorithm.** A pure-diversity (representativeness) acquisition: it picks a batch that best covers the unlabeled pool in a frozen foundation-model embedding space, with no use of model uncertainty or labels. Each pool/labeled image is embedded once with the SAM (vit_h) image encoder and reduced to a 256-D vector; selection is the greedy k-center (farthest-first / robust k-center) heuristic of Sener-Savarese, seeded from the already-labeled set. It iteratively adds the pool point whose distance to its nearest already-selected/labeled center is largest, minimizing the maximum coverage radius.

**Acquisition signal.** k-center-greedy in SAM feature space. Build feature matrix over [labeled; pool] (optionally L2-normalized rows). For k iterations, select argmax over unselected pool points of the minimum distance to the current center set (labeled ∪ already-chosen): u* = argmax_{u in pool} min_{c in centers} dist(f(u), f(c)), where dist defaults to L2 ("l2"/euclidean) and f(x) = global-average-pooled SAM vit_h image-encoder feature (256-D). No score() function — there is no per-sample uncertainty signal; selection is entirely the greedy coverage criterion. Returns k pool indices.

**Implementation.** policies/p7_sam_coreset.py (class SAMCoreSet, @register("P7"), name "Foundation-CoreSet", needs_pred_cache=False, needs_features=("foundation",)). select() reads ctx.features["foundation_pool"] and ctx.features["foundation_label"] (asserts both present), optionally L2-normalizes each row via _l2_norm (default normalize=True; clips norm at 1e-12), concatenates [label_feats; pool_feats] so the labeled rows occupy indices [0, n_label), computes a full dense pairwise distance matrix with sklearn.metrics.pairwise_distances(metric=self.metric, default "l2"), then calls _helpers.kcenter_greedy(dist_mat, init_idx=range(n_label), k=k). kcenter_greedy (policies/_helpers.py) is plain greedy farthest-first: marks init_idx as labeled, asserts init non-empty, and each step takes dist_mat[~labeled][:,labeled].min(axis=1).argmax() as the new center. Returned global indices are mapped back to pool indices via (i - n_label), dropping any < 0 (i.e. excluding labeled). Features are precomputed upstream, not in this file: features/sam.py extract_sam_features runs the frozen SAM image encoder -> (B,256,64,64) and does global average pool over the 64x64 grid -> (B,256) per image (SAM_POOLING_RULE="global_avg_pool_over_64x64", FEATURE_DIM=256 for all ViT sizes); runner/precompute_sam.py warms/caches these to HDF5 for each (dataset, seed) pool with --sam-model-type vit_h, and the runner places them into ctx.features as "foundation_pool"/"foundation_label". The encoder_id/checkpoint/pooling_rule/feature_layer/cache_version are logged in the trajectory's foundation field (constraint #8).

**Caveats.** Requires precomputed SAM-H / vit_h foundation features (foundation_pool + foundation_label); select() asserts both are present and there is no fallback (no SAM cache => hard failure). Canonical extractor is vit_h (sam_vit_h_4b8939.pth) per profiles/frozen.py and STATE.md; the in-file docstring's "ViT-B" is stale and must not be cited — vit_b selections differ materially (pilot overlap 0.19-0.38) and never fall back silently. RAM/compute: builds a full dense (n_label+n_pool)^2 pairwise distance matrix in memory, so it scales quadratically with pool size (mitigated by the pool_cap used in precompute). Deviates from Sener-Savarese in two documented, deliberate ways: (1) features are L2-row-normalized by default (normalize=True) rather than raw — a deliberate choice consistent across the CoreSet family P3/P5/P7 (stage1p5 review); (2) k-center is seeded from the labeled set (init_idx=range(n_label)), and kcenter_greedy asserts a non-empty init, so a cold-start (zero labeled) config fails loudly rather than under-selecting. Uses plain greedy k-center (no robust/MIP outlier variant). One-shot encode: SAM features are extracted once up front and cached, never recomputed per round.

**File.** `medal_bench/policies/p7_sam_coreset.py`

## P8 — Foundation-TypiClust (TypiClust on SAM features)  _(diversity)_

**Paper.** Guy Hacohen, Avihu Dekel, Daphna Weinshall. "Active Learning on a Budget: Opposite Strategies Suit High and Low Budgets." Proceedings of the 39th International Conference on Machine Learning (ICML), PMLR 162, 2022. (Reference implementation: github.com/avihu111/TypiClust.) Here adapted to slice-level medical segmentation by clustering on frozen SAM ViT-H image-encoder features instead of self-supervised features.

**Algorithm.** TypiClust is a low-budget, diversity-based cold-start acquisition. It clusters all samples (labeled union unlabeled) in feature space into n_clusters = |L| + budget, treats clusters containing a labeled sample as "covered" and the rest as "uncovered," and queries the most TYPICAL (highest local density, i.e. not an outlier) unlabeled point from under-covered clusters first. This combines coverage/diversity (one query per cluster, uncovered-first) with a preference for dense, representative points over outliers. Here it operates on frozen SAM ViT-H image features rather than the paper's self-supervised features; it reads no ground-truth masks.

**Acquisition signal.** Per unlabeled sample i in cluster c: typicality(i) = 1 / (mean L2 distance to its K nearest in-cluster neighbours + 1e-12), with K = min(knn=20, max(1, |cluster|//2)). Selection is not a flat top-k by this score: clusters are ordered uncovered-first then by descending size, and the budget is filled ROUND-ROBIN, taking the next-most-typical unlabeled point from each eligible cluster in turn. A cluster is eligible only if its TOTAL (L union U) membership exceeds min_cluster_size=5 (drops outlier/singleton clusters); higher typicality = denser/more representative point preferred.

**Implementation.** In medal_bench/policies/p8_sam_typiclust.py (class FoundationTypiClust, no score() override — selection is feature-clustering only). select() reads ctx.features["foundation_pool"] (N,256) and ctx.features["foundation_label"] (the SAM-H per-image features computed and cached by medal_bench/features/sam.py: replicate grayscale->3ch, resize 1024x1024, SAM normalize, ViT-H image encoder -> (256,64,64), global-avg-pool -> 256-D). Steps: (1) stack X = [label; pool], L2-normalize rows (normalize=True); (2) n_clusters = max(1, min(n_lab + k, max_clusters=500, n_total)); KMeans(n_clusters, random_state = ctx.query_seed or ctx.seed+ctx.round_idx, n_init=10).fit_predict; (3) covered = clusters appearing in the first n_lab rows; build cluster->pool-index map; (4) _typicality() computes per-point 1/mean-KNN-distance via sklearn pairwise_distances within each cluster, K-capped to min(knn, len//2); (5) eligible = non-empty clusters with bincount membership > min_cluster_size (graceful relax to all non-empty if none qualify); (6) order = sorted uncovered-first then by descending cluster size; (7) round-robin pop most-typical-first from each cluster's ranked list until k filled; last-resort global-typicality fill if eligible members are exhausted; (8) map combined indices back to pool indices (i - n_lab); rich diagnostics (n_clusters, num_filtered/singleton clusters, n_uncovered, selected_from_uncovered, relaxed flag) written to ctx.diagnostics_out. Helper files: medal_bench/features/sam.py (feature cache, shared with P7); medal_bench/policies/base.py (Policy/PolicyContext interface). It does NOT use _helpers.py.

**Caveats.** Requires precomputed SAM-H / vit_h foundation features (asserts foundation_pool present; needs_features=("foundation",), needs_pred_cache=False); the SAM image-encoder feature extraction is the cost driver and the same cache is shared with P7. This P8 is the reference-faithful variant with MIN_CLUSTER_SIZE filter + round-robin + K-cap=len//2; the simplified unlabeled-only/cosine-density/n_clusters=k version is the ablation-only P8b (SAM-DensityClust, is_ablation=True) and the pre-v3 single-pass + global-fallback behaviour is deprecated ablation P8c — do not conflate them. If k >= n_pool it returns the whole pool. TypiClust is a low-budget cold-start method by design; like other diversity policies it carries no per-round uncertainty signal.

**File.** `medal_bench/policies/p8_sam_typiclust.py`

## P9 — PAAL (Predictive Accuracy-Based Active Learning)  _(predictor)_

**Paper.** Jun Shi, Shulan Ruan, Ziqi Zhu, Minfan Zhao, Hong An, Xudong Xue, Bing Yan. "PAAL: Predictive Accuracy-Based Active Learning for Medical Image Segmentation." Proceedings of the Thirty-Third International Joint Conference on Artificial Intelligence (IJCAI-24), Main Track, pp. 4885-4893, 2024. Reference implementation: shijun18/PAAL-MedSeg (GitHub). NOTE: the in-file docstrings attribute the method to "Yi et al."; the canonical IJCAI 2024 paper's lead/corresponding authorship is Jun Shi et al. (the reference repo shijun18/PAAL-MedSeg matches). The DOI/proceedings entry should be treated as authoritative over the docstring's "Yi et al." shorthand.

**Algorithm.** PAAL is a predictor-based AL method. An auxiliary Accuracy Predictor (AP) network is trained on the labeled set to regress the segmentation model's per-class Dice from the image and the seg softmax probabilities. The unlabeled pool is then scored by how LOW the AP thinks the seg accuracy will be (low predicted accuracy = high acquisition priority). Final selection uses a Weighted Polling Strategy (WPS): cluster the pool by task-UNet features, sort each cluster by score, and round-robin across clusters until the budget is filled (combining informativeness with diversity). This is the fixed-budget PAAL variant; the paper's Incremental Querying (IQ) is intentionally NOT enabled to keep rounds comparable with other policies.

**Acquisition signal.** Per-sample acquisition score (higher = selected first): with AP-predicted per-class accuracy hat-a in [0,1] (clamped to [eps, 1-eps]), score = -mean_c log(hat-a_c) over the scored class set (default score_mode="neg_log_mean_acc"; an alternative one_minus_mean_acc = 1 - mean_c hat-a_c exists). Default include_background=True means the mean is over ALL C output channels (matching the official mean(axis=1)); include_background=False restricts to foreground classes 1..C-1. AP training target is per-class HARD argmax-Dice T(P, mask) in [0,1] with MSE loss; a class absent from both prediction and GT for a sample scores Dice=1.0. Selection over scores is WPS (KMeans on task-UNet features + round-robin), not plain top-k.

**Implementation.** In p9_paal.py (class PAAL, @register("P9"), needs_pred_cache=True, needs_features=("task_unet",)). score(): (1) build/warm-start the AP (AccuracyPredictor from _paal_ap.py, a verbatim ResNet-18 BasicBlock[2,2,2,2] backbone with sigmoid per-class head; in_ch = image_channels + num_classes). persist_ap_weights=True carries AP weights across AL rounds within a (dataset,seed) cell (re-init only on signature change of (image_channels,num_classes)); seeded by torch.manual_seed(seed+round_idx). (2) _train_ap(): seg model frozen/eval; for each labeled batch (>=2 samples; BatchNorm needs >1) compute probs=softmax(model(img)) and target via hard_dice_per_class() from _paal_ap.py, feed AP input = cat([img, probs], dim=1), MSE(pred, dice_tgt) over all classes, AdamW lr=1e-3, 200 epochs by default. A held-out labeled split gives the paal_ap_val_corr Pearson diagnostic. (3) Pool scoring: batch over ctx.pool, seg probs taken from ctx.pred_cache.probs (not recomputed), AP forward -> per-class predicted accuracy, clamp to [eps,1-eps], reduce to per-sample score via score_mode. select() implements WPS: pull ctx.features["task_unet_pool"], L2-normalize, KMeans with n_clusters = int(log2(max(4k,2))+1) clamped to [1, min(n,k)] (random_state=seed+round_idx, n_init=10), sort each cluster by score desc, round-robin pop until k selected. Helper file: _paal_ap.py (AccuracyPredictor + hard_dice_per_class). _helpers.py is NOT used by P9. Optional speedup MEDAL_P9_CACHE_AP=1 precomputes the frozen-seg probs/Dice once instead of every epoch (off by default).

**Caveats.** RAM/compute: P9 trains a full ResNet-18 AP from scratch every AL round and feeds the full (N,C,H,W) probs through a network (needs_pred_cache_probs=True), so on high-class-count datasets it is memory- and compute-heavy; the per-epoch seg-forward recompute is the dominant cost on slow GPUs and previously blew past the 4h cell timeout (mitigated by the opt-in MEDAL_P9_CACHE_AP=1 cache). Low-budget predictor failure: the AP is trained only on the small labeled set, so in early/micro-budget rounds it has too few samples to learn a reliable accuracy regressor; the project's audit found P9 PAAL to be a genuine on-theory micro-budget failure (no implementation bug) where the predictor underperforms simple uncertainty/random at small budgets. Faithfulness: this is the fixed-budget variant only (paper's Incremental Querying / IQ is deliberately disabled). The frozen.py-pinned task_unet features are required for WPS clustering (assert fails without task_unet_pool). Docstring author attribution ("Yi et al.") disagrees with the canonical IJCAI-24 author list (Shi et al.) — trust the proceedings citation.

**File.** `medal_bench/policies/p9_paal.py`
